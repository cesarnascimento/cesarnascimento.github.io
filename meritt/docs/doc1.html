Entendendo o front-end

Login

O login é feito fazendo uma requisição para o backend e armazenando o access token no localStorage. Esse access token é utilizado posteriormente para fazer outras requisições de API.

Sign Up

A tela de cadastro segue o mesmo fluxo do login e cadastra o usuário pelo backend.

Homepage

A tela de homepage é a página principal que contem o menu e todos os outros componentes. Ela utiliza o hook useEffect para checar se o usuário está logado e pegar possíveis itens do backend. Se a requisição falhar, o usuário é deslogado do app.

Admin

Existem 2 tipos de usuário. O padrão, que é o estudante e o administrador. Quando logado como um administrador(user: adm, senha: 123456), uma nova opção aparece no menu e é possível acessar essa rota privada. Caso o usuário não tenha permissão para essa rota, ele é jogado para a página inicia.

Demais componentes

Os componentes mais simples foram feitos com styled componentes e material ui. Testes unitários foram implementados mas não foi possível fazer testes mais aprofundados.
